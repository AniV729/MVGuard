    What happens on registration
    1. Staff account created with hashed password (Werkzeug PBKDF2-SHA256)
    2. A unique UOV key pair is generated server-side:
      • Public key → stored in DB, used to encrypt patient records
      • Private key → stored in DB (in production: user-key-wrapped)
    3. Key generation takes a few seconds (GF(256) matrix operations)
    4. Role determines decryption access: only Doctors and Admins can call the decrypt endpoint
